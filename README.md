# 거래처별 업무 관리 시스템

거래처별 반복 업무를 등록, 조회, 완료 처리, 출력할 수 있는 정적 업무 관리 앱입니다. 외부 라이브러리 없이 순수 HTML5, CSS3, Vanilla JavaScript만으로 동작합니다.

## 주요 기능

- 회사명 입력 및 앱 제목 반영
- 거래처 등록, 변경, 삭제, 조회
- 거래처명 또는 사업자등록번호 검색 기반 중복 등록 방지
- 업무 현황 상태별 집계
- 상태 박스 클릭 시 해당 조건의 업무 리스트로 이동
- 신규 업무 생성
- 매월 반복 업무 등록
- 반복 업무 완료 시 다음 회차 업무 자동 생성
- 업무 선택 후 수정, 삭제, 완료 처리
- 시작일, 종료 예정일, 완료일 기준 검증
- 종료 예정일 기준 기간 조회
- 조회/출력 테이블, 인쇄, CSV 다운로드
- UTF-8 BOM 포함 CSV 생성으로 엑셀 한글 깨짐 방지
- 사용자 입력값 `textContent` 렌더링을 통한 XSS 방어

## 상태 기준

업무 상태는 화면에서 아래 기준으로 계산됩니다.

- `완료`: 시작일과 완료일이 모두 있는 업무
- `진행 중`: 시작일만 있고 완료일이 없는 업무
- `경과`: 시작일이 없고 종료 예정일이 오늘 이전인 업무
- `당일`: 시작일이 없고 종료 예정일이 오늘인 업무
- `예정`: 시작일이 없고 종료 예정일이 D+1일부터 D+3일인 업무
- `미도래`: 시작일이 없고 종료 예정일이 D+4일 이후인 업무

업무 현황에는 미완료 업무 중심으로 `경과`, `진행 중`, `당일`, `예정`, `미도래`만 표시합니다. `완료`는 업무 리스트와 조회/출력 필터에서 확인합니다.

## 반복 업무

신규 업무 생성 시 `매월 반복 업무로 등록`을 선택할 수 있습니다.

반복 업무를 완료 처리하면 같은 거래처, 같은 업무 내용, 다음 달 동일 기준일의 업무가 자동 생성됩니다.

- 이미 같은 반복 그룹의 다음 회차가 있으면 중복 생성하지 않습니다.
- 이미 같은 거래처, 업무 내용, 종료 예정일의 업무가 있으면 중복 생성하지 않습니다.
- 반복 업무의 종료 예정일을 수정하면 다음 회차 기준일도 수정된 날짜 기준으로 갱신됩니다.
- 완료된 반복 업무를 단순 수정할 때는 다음 회차가 추가 생성되지 않습니다.

## 날짜 입력

화면 날짜 입력과 표시는 `YY-MM-DD` 형식을 사용합니다.

내부 저장은 `YYYY-MM-DD` 형식으로 정규화합니다.

완료 처리 시 아래 검증을 수행합니다.

- 시작일은 오늘보다 늦을 수 없습니다.
- 완료일은 오늘보다 늦을 수 없습니다.
- 완료일은 시작일과 같거나 이후여야 합니다.
- 완료일을 삭제하고 시작일만 남기면 상태는 `진행 중`으로 표시됩니다.
- 시작일과 완료일이 모두 없으면 종료 예정일 기준 상태로 표시됩니다.

## 실행 방법

브라우저에서 `index.html` 파일을 직접 열면 실행됩니다.

별도 설치, 빌드, 개발 서버가 필요하지 않습니다.

## 데이터 저장소

브라우저 LocalStorage에 아래 key로 저장됩니다.

```text
sme_b2b_system_data
```

저장 구조는 다음과 같습니다.

```json
{
  "companyName": "",
  "clients": [
    {
      "id": "",
      "name": "",
      "businessNumber": "",
      "contactName": "",
      "phone": "",
      "email": "",
      "createdAt": "",
      "notes": ""
    }
  ],
  "tasks": [
    {
      "id": "",
      "clientId": "",
      "title": "",
      "description": "",
      "startDate": "",
      "dueDate": "",
      "status": "waiting",
      "completedAt": "",
      "completionNote": "",
      "executionMemo": "",
      "recurrence": "",
      "recurrenceGroupId": "",
      "recurrenceDay": 0
    }
  ]
}
```

`status`는 내부 저장 코드로 `waiting`, `active`, `complete`를 사용하며 화면에는 한글 상태명으로 표시됩니다. 기존 한글 저장값은 로드 시 자동 변환됩니다.
