# 구현 프롬프트

외부 라이브러리 없이 순수 HTML5, CSS3, Vanilla JavaScript ES6+만 사용하여 `index.html`, `style.css`, `app.js` 파일로 구성된 `거래처별 업무 관리 시스템`을 구현한다.

## 기본 조건

- React, jQuery, Tailwind CSS, Vite 런타임을 사용하지 않는다.
- 앱은 브라우저에서 `index.html`을 직접 열어 실행 가능해야 한다.
- 사용자 입력값을 화면에 표시할 때 `innerHTML`을 사용하지 않고 DOM 생성과 `textContent`를 사용한다.
- 날짜 입력과 화면 표시는 `YY-MM-DD` 형식을 사용한다.
- 내부 저장 날짜는 `YYYY-MM-DD` 형식으로 정규화한다.

## 데이터 저장

- LocalStorage key는 `sme_b2b_system_data`를 사용한다.
- 저장 구조는 아래 형식을 기본으로 한다.

```json
{
  "companyName": "",
  "clients": [],
  "tasks": []
}
```

- 최초 진입 시 샘플 거래처와 샘플 업무를 생성한다.
- task 객체에는 아래 필드를 포함한다.

```json
{
  "id": "",
  "clientId": "",
  "title": "",
  "description": "",
  "startDate": "",
  "dueDate": "",
  "status": "waiting",
  "completedAt": "",
  "completionNote": "",
  "executionMemo": "",
  "recurrence": "",
  "recurrenceGroupId": "",
  "recurrenceDay": 0
}
```

- 내부 `status` 값은 `waiting`, `active`, `complete` 코드를 사용한다.
- 기존 한글 저장값 `대기`, `진행중`, `진행 중`, `완료`는 로드 시 코드값으로 변환한다.

## 화면 구성

단일 HTML 파일 안에서 nav 클릭 시 section의 `active` 클래스를 전환하여 화면을 바꾼다.

메뉴는 아래 4개로 구성한다.

- 업무 현황
- 거래처 관리
- 업무 리스트
- 조회/출력

반복업무 템플릿 메뉴는 사용하지 않는다.

## 앱 제목

- 기본 제목은 `거래처별 업무 관리 시스템_(회사명)` 형식이다.
- 회사명은 입력 가능해야 한다.
- 회사명이 비어 있으면 `회사명`을 표시한다.

## 거래처 관리

- 거래처명 또는 사업자등록번호를 2글자 이상 검색한 뒤 등록/변경/삭제/조회 영역을 표시한다.
- 거래처 등록 필드:
  - 거래처명 필수
  - 사업자등록번호 선택
  - 담당자 필수
  - 연락처 필수
  - 이메일 선택
  - 등록일 자동
  - 메모 선택
- 거래처명과 사업자등록번호 중복 등록을 방지한다.
- 검색어와 중복 가능 거래처명을 화면에 표시한다.

## 업무 현황

상태 박스는 아래 5개만 표시한다.

- 경과
- 진행 중
- 당일
- 예정
- 미도래

완료 박스는 업무 현황에는 표시하지 않는다.

상태 박스는 숫자만 굵게 표시한다.

상태 박스 hover 시 15% 확대하고 배경색은 `#00C0E8`로 변경한다.

상태 박스 클릭 시 업무 리스트로 이동하고 해당 상태 필터를 적용한다.

## 상태 기준

화면 상태는 저장된 `status` 값이 아니라 날짜 필드로 계산한다.

- 완료: 시작일과 완료일이 모두 있는 업무
- 진행 중: 시작일만 있고 완료일이 없는 업무
- 경과: 시작일이 없고 종료 예정일이 오늘 이전인 업무
- 당일: 시작일이 없고 종료 예정일이 오늘인 업무
- 예정: 시작일이 없고 종료 예정일이 D+1일부터 D+3일인 업무
- 미도래: 시작일이 없고 종료 예정일이 D+4일 이후인 업무

## 업무 리스트

- 기본 화면은 업무 조회 중심으로 구성한다.
- 신규 업무 생성 폼은 기본 숨김 처리하고 `신규 업무` 버튼 클릭 시 표시한다.
- 신규 업무 필드:
  - 거래처명
  - 종료 예정일
  - 업무 내용
  - 매월 반복 업무 여부
- 신규 업무 생성 시 `거래처 + 업무 내용 + 종료 예정일` 기준 중복을 방지한다.
- 업무 조회 조건:
  - 상태 유형: 전체, 경과, 진행 중, 당일, 예정, 미도래, 완료
  - 종료 예정일 기준 기간 From, To
  - 기본 기간: 오늘 D-1month부터 D+1month
  - 기간 범위 제한: 최대 1년
- 조회 버튼 클릭 시 업무 목록을 표시한다.
- 업무 목록 체크박스를 선택하면 업무 처리 모달을 연다.

## 업무 처리 모달

모달에는 아래 필드를 둔다.

- 거래처명
- 업무 내용
- 종료 예정일
- 시작일
- 완료일
- 완료 내용
- 실행 메모
- 삭제
- 수정
- 취소
- 완료

검증 규칙:

- 종료 예정일은 필수이다.
- 시작일은 오늘보다 늦을 수 없다.
- 완료일은 오늘보다 늦을 수 없다.
- 완료일은 시작일과 같거나 이후여야 한다.
- 완료 버튼을 누를 때 완료일이 비어 있으면 오늘 날짜를 완료일로 저장한다.
- 완료 버튼을 누를 때 시작일이 비어 있으면 완료일과 같은 날짜를 시작일로 저장한다.
- 완료일을 삭제하고 시작일만 저장하면 상태는 진행 중으로 표시한다.
- 시작일과 완료일이 모두 없으면 종료 예정일 기준 상태로 표시한다.
- 완료일이 없는 상태로 저장하면 완료 내용과 실행 메모도 비운다.

## 매월 반복 업무

- 신규 업무 생성 시 `매월 반복 업무로 등록`을 선택하면 task에 `recurrence: "monthly"`를 저장한다.
- 반복 업무에는 `recurrenceGroupId`와 `recurrenceDay`를 저장한다.
- 반복 업무가 미완료에서 완료로 전환될 때 다음 회차 업무를 자동 생성한다.
- 완료된 반복 업무를 단순 수정할 때는 다음 회차를 추가 생성하지 않는다.
- 다음 회차 생성 기준:
  - 같은 거래처
  - 같은 업무 내용
  - 다음 달 동일 기준일
  - 시작일과 완료일은 비워 둔다.
  - 상태는 내부 코드 `waiting`으로 저장한다.
- 다음 회차 중복 방지:
  - 같은 반복 그룹과 같은 종료 예정일이 이미 있으면 생성하지 않는다.
  - 같은 거래처, 업무 내용, 종료 예정일의 업무가 이미 있으면 생성하지 않는다.
- 반복 업무의 종료 예정일을 수정하면 `recurrenceDay`도 수정된 종료 예정일 기준으로 갱신한다.
- 월말 반복은 해당 월의 마지막 날짜를 넘지 않도록 보정한다.

## 조회/출력

- 조회/출력 상태 필터는 전체, 경과, 진행 중, 당일, 예정, 미도래, 완료를 제공한다.
- 검색은 거래처명 또는 업무 내용을 대상으로 한다.
- 종료 예정일 기준 기간 From, To를 제공한다.
- 기본 기간은 오늘 D-1month부터 D+1month이다.
- 기간 범위는 최대 1년으로 제한한다.
- 표에는 아래 열을 표시한다.
  - 거래처
  - 업무 내용
  - 종료 예정일
  - 상태
  - 완료일
  - 완료 내용
  - 실행 메모
- 인쇄 버튼은 `window.print()`를 호출한다.
- 인쇄 시 내비게이션, 액션 버튼, 모달은 숨기고 조회 테이블 중심으로 출력한다.
- CSV 다운로드 시 UTF-8 BOM `\uFEFF`를 파일 앞에 붙인다.
- CSV 문자열 셀은 큰따옴표로 감싸고 내부 큰따옴표는 `""`로 이스케이프한다.

## 검증

- `node --check app.js`가 통과해야 한다.
- 아래 검색어가 앱 파일에 남지 않아야 한다.

```bash
rg "innerHTML|src/main|react|tailwind|jquery|templates|업무명|마감일|실제완료일|완료 날짜|완료내용|실행메모" index.html app.js style.css
```

- 신규 업무 생성, 반복 업무 완료, 다음 회차 자동 생성, 상태 필터, 조회/출력, CSV 다운로드를 브라우저에서 확인한다.
